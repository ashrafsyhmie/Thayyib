# AGENTS.md — Instructions for AI Coding Assistants

## Required Behavior

Before coding:
1. Read `MASTER_CONTEXT.md`
2. Read relevant files in `context/`
3. Summarize understanding
4. Propose plan
5. Then implement

## Project Identity

This is a serious FYP-to-startup project.

Write code as if this may become a real SaaS product.

## Development Rules

- Keep implementation beginner-friendly
- Prefer clear code over clever code
- Do not introduce unnecessary complexity
- Use strong typing
- Validate inputs
- Reuse components
- Keep AI outputs auditable
- Store AI results with sources and confidence
- Add comments only where useful

## Do Not

- Do not build random features outside scope
- Do not overengineer microservices early
- Do not add blockchain unless specifically requested
- Do not claim AI decisions are final compliance rulings
- Do not replace human compliance officers

## AI Safety Rule

AI analysis must be framed as assistance.

The system should say:
"Potential risk detected. Please verify with a qualified halal compliance officer."

Not:
"This product is halal/non-halal."
